## Adobe Service Runtime
Adobe Service Runtime or ASR is an implementation of the Microservice chassis pattern. It provides mechanisms to handle cross cutting concerns like logging, health checks, metrics, easy access to foundational Adobe services via Connectors, etc.

## Why is ASR required?
1. Provide a collection of components, tools, and best practices (12 factor) which enable rapid development of four 9s capable innovative and secure services at Adobe by product and research teams 
2. Enable a container first deployment system

## My Role in this Project

I am the Hands On Lead Engineer/Architect/Product Manager for this suite of Libraries

## Impact of ASR at Adobe

- Using ASR has saved 4-6 weeks of setup time for new Java Projects at Adobe
- Secure BOM for every Adobe service that uses ASR
